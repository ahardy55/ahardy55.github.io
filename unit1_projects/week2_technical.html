ID's Vs. Classes

One of the fun things about when first learning HTML is nervously waiting for someone to point out that you're using IDs and Classes wrong. They both seem to serve the same function: to isolate an element for CSS styling, yet seem arbitrability applied. What is the difference? Why do two separate ways to ID an element exist to begin with? When should ID be used instead of Class? Well, fear not HTML/CSS novice, for I will answer these questions!

Classes and ID's can be added to HTML elements, allowing you to select and manipulate them in CSS. Have a photo you want sized differently from your other images? Give it an ID:

HTML
<img src="photo.jpg" id="bigPhoto">

CSS
#bigPhoto {
    size: 100px;
}

But what if you have a group of photos that you want to be the same size? That is where class comes into play.

HTML
<img src="photo1.jpg" class="smallPhoto">
<img src="photo2.jpg" class="smallPhoto">
<img src="photo3.jpg" class="smallPhoto">

CSS
.smallPhoto {
   size: 30px;
}

What's the difference? Classes can be used on multiple elements, while IDs are used on single elements. For example, since your page will only have one header, it would make more sense to give that header an ID instead of a class. Classes can be used on multiple elements. If you want to style multiple elements the same way, give them the same class. An element can also contain an ID and a class. Or multiple classes...but we you can learn about that and when they should be used from someone a bit more knowledgable on the subject than me!

IDs also serve another important function. Click this link. See, it took you directly to this paragraph instead of just refreshing the page! That's because I gave this paragraph an ID, and added the ID hash onto the end of the URL like so: http://ahardy55.github.io/unit1_projects/week2_technical.html#idHash. Your browser will attempt to find the section with that ID, which is why your IDs absolutely must be unique.

There you have it: Use a class when you need it for multiple elements, use an ID when it applies to only a single element. And if you don't either need them, don't use them. Adding a lot of useless classes and IDs will do nothing but increase the likelihood of messing up your page.
